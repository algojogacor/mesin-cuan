# Contributing to mesin-cuan

Thank you for your interest in contributing. This document explains how to get started.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b feature/your-feature`)
3. **Make your changes** — keep them focused and small
4. **Test** your changes locally
5. **Push** and open a **Pull Request**

## Pull Request Guidelines

- Write a clear title and description
- Reference any related issues (`Closes #X`)
- Keep PRs small — one feature or fix per PR
- Update documentation if your change affects how the project is used
- Use the [PR template](.github/PULL_REQUEST_TEMPLATE.md) if available

## Code Style

- Follow existing code patterns in the project
- Write clear variable names
- Add comments for complex logic
- Keep functions focused and reasonably sized

## Reporting Issues

Found a bug? Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, Python version, etc.)

## Getting Help

Stuck? Open a [discussion](https://github.com/algojogacor/mesin-cuan/discussions) or comment on your PR — we are happy to help.
